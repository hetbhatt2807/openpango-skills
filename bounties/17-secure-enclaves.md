## 📌 Description
Standard Docker isolation is fundamentally insufficient for executing wildly unpredictable, AI-generated untrusted code. We require true kernel-level sandboxing. The "Secure Enclave" must either be an integrated AWS Firecracker MicroVM orchestrator or a custom eBPF (Extended Berkeley Packet Filter) kernel module.

### ✅ Requirements
- **Hyper-Isolation via MicroVMs**: Build an orchestration layer in Rust or Go that provisions and boots AWS Firecracker MicroVMs for every single agent task.
- **Sub-50ms Cold Starts**: Implement rootfs cloning, snapshotting, and memory deduplication (KSM) to ensure the MicroVMs can boot to a ready-state python environment in less than 50 milliseconds.
- **eBPF Syscall Interception**: If the eBPF path is chosen over Firecracker, write a custom kernel module that dynamically hooks and drops specific syscalls (like `execve`, `ptrace`, or raw socket creation) based on a dynamic YAML capabilities manifest generated by the Orchestrator.
- **Deterministic Record & Replay**: The sandbox must capture execution traces to allow for deterministic playback (using mechanisms akin to Mozilla's `rr` tool). This is strictly required to debug non-deterministic AI agent hallucinations.
